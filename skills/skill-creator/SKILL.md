# スキルクリエイター

ワークスペースの既存スキル群と統一感のあるスキルを作成・改善する。

## 絶対遵守事項

- 対話は日本語で行う
- 作成するスキルのSKILL.mdも日本語で記述

## スキルとは

スキルは「AIへの指示書」。SKILL.mdファイルに手順を書いておくと、AIがその通りに動いてくれる。

**メリット:**
- 毎回同じ説明をしなくていい
- 複雑な手順を自動化できる
- チームで共有できる

## スキルの4分類

| 分類 | 特徴 | 例 |
|------|------|-----|
| 対話型 | ユーザーとの対話で成果物を生成 | ブログ記事生成、プレゼン企画 |
| 自動実行型 | 指示一つで取得→加工→出力 | ニュース取得、サイト巡回 |
| ツール連携型 | 外部ツール/APIとの連携 | YouTube字幕取得、API操作 |
| 思考支援型 | 構造化された思考フレームワーク提供 | ブレスト支援、調査整理 |

## スキル作成フロー

### Step 1: ヒアリング

ユーザーに以下を確認（わかっている情報はスキップ）：

1. **何をするスキル？** → 主機能の把握
2. **どんな時に使う？** → トリガー条件の特定
3. **入力と出力は？** → データフローの明確化
4. **スクリプトは必要？** → Python/Bash等の自動化有無

### Step 2: 分類と設計

4分類のどれに該当するか判定し、適切な構成を選択。

### Step 3: ディレクトリ作成

```bash
# スキル名はケバブケースで
mkdir -p [WORKSPACE]/skills/<skill-name>

# 必要に応じてサブディレクトリ作成
mkdir -p [WORKSPACE]/skills/<skill-name>/scripts
mkdir -p [WORKSPACE]/skills/<skill-name>/references
```

### Step 4: スクリプト作成（該当する場合）

スクリプトが必要な場合：
1. `scripts/` にPython/Bashスクリプトを配置
2. `uv run` で実行できるようにする
3. `--help` オプションを実装
4. **実際にテスト実行して動作確認**

### Step 5: SKILL.md作成

以下のテンプレートを使う：

```markdown
---
name: <skill-name>
description: [50-200文字。何をするか＋いつ使うか＋トリガーフレーズ]
---

# [スキル名]

[1行の説明]

## 絶対遵守事項（必要な場合）

[このスキル固有の制約]

## 手順

### Step 1: [最初のステップ]

[具体的な作業内容]

### Step 2: [次のステップ]

[具体的な作業内容]

### Step 3: [出力]

[出力フォーマット]

## 使用例

[呼び出し例]
```

**パス規則:**
- `[SKILL_DIR]` - SKILL.mdがあるディレクトリ
- `[WORKSPACE]` - ワークスペースルート
- ハードコードパス禁止

**保存機能がある場合:**
- note-takingスキルがあればそちらへの委譲を記述
- 直接保存ロジックを書かない

### Step 6: 品質チェック

以下をすべて確認：

- [ ] name と description がフロントマターにある
- [ ] description は50-200文字で、トリガーフレーズを含む
- [ ] 手順がStep形式で明確
- [ ] コマンドがある場合は具体例つき
- [ ] ハードコードパスがない（`[WORKSPACE]`, `[SKILL_DIR]` を使用）
- [ ] 出力フォーマットが定義されている
- [ ] 使用例がある

### Step 7: 登録

スキルの配置場所がシンボリックリンクで管理されている場合（例: `.claude/skills/` → `skills/`）、`skills/` にフォルダを作るだけで自動認識される。

そうでない場合は、AIツールの設定に新しいスキルを登録する。

### Step 8: README更新

`[WORKSPACE]/skills/README.md` があれば、スキル一覧に追加。

### Step 9: 完了報告

```
スキル作成完了！

skills/<skill-name>/
- SKILL.md（XX行）
- scripts/（あれば）
- references/（あれば）
```

## スキル改善フロー

既存スキルの改善を依頼された場合：

### Step 1: 現状分析

対象スキルのSKILL.mdを読み、構造と内容を把握する。

### Step 2: 品質チェック適用

Step 6のチェックリストで問題点を洗い出し。

### Step 3: 改善実施

問題点を修正し、品質チェックを再実行。

### Step 4: 完了報告

変更箇所をまとめて報告。

## 良いスキルを作るコツ

### descriptionは具体的に

**良い例:**
```
YouTubeの動画URLから字幕を取得してノートにまとめる。「YouTube動画をまとめて」で使用。
```

**悪い例:**
```
動画をまとめる
```

### 手順は明確に

- Step形式で番号付き
- コマンドがあれば具体的に書く
- 曖昧な表現を避ける

### 出力フォーマットを決める

期待する出力の形を示しておくと、毎回同じ品質になる。

### 既存スキルを参考にする

ワークスペース内の既存スキルを読んで、統一感のある構成にする。

```bash
# 既存スキルの一覧
ls [WORKSPACE]/skills/

# 参考になりそうなスキルを読む
cat [WORKSPACE]/skills/<similar-skill>/SKILL.md
```

## 使用例

```
新しいスキルを作りたい
```
